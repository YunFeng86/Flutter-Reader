## Why

Fleur 已经具备跨平台运行能力和基础自适应布局，但当前 UI 仍然主要停留在默认 Material 3 语气与局部样式修补阶段，缺少能够覆盖桌面与移动端的统一主题系统、场景级版式语言和一致的交互反馈。随着项目逐步面向 Windows、Android、macOS、Linux 与 iOS 等多平台维护，继续依赖零散组件样式会放大视觉不一致、实现重复和后续迭代成本。

## What Changes

- 建立一套跨平台 UI 基础能力，包括主题 token、组件主题归一化、语义 surface/state token，以及桌面/移动端的密度与交互差异策略。
- 重构首页导航、侧栏、文章列表与阅读器的布局和视觉层级，使其从“默认组件组合”升级为“专业阅读工作台”。
- 为阅读器引入独立场景主题，统一标题区、摘要卡、正文容器、搜索条、底部工具条等元素的版式和表面语义。
- 引入一套克制但一致的动效基线，用于列表显现、状态胶囊、浮层、导航与阅读场景切换中的反馈与过渡。
- 保持全平台单一产品身份，不为每个平台复制一套皮肤，而是在统一品牌骨架之上尊重桌面与移动端的交互习惯。

## Capabilities

### New Capabilities
- `adaptive-ui-foundation`: 定义 Fleur 的跨平台主题系统、语义 token、组件主题归一化和导航/侧栏/列表的基础布局规则。
- `reader-scene-presentation`: 定义阅读器作为独立场景的版式、表面语义、工具条和搜索浮层展示规则。
- `motion-feedback-system`: 定义跨平台 UI 中允许使用的显现、切换、状态反馈与场景过渡动效基线。

### Modified Capabilities

None.

## Impact

- 受影响代码主要位于 `lib/theme/`, `lib/widgets/`, `lib/ui/`, `lib/screens/` 与设置接线相关代码。
- 需要新增或重组主题扩展、组件主题与阅读器局部主题装配逻辑。
- 需要补充与更新桌面/移动端布局、交互和视觉回归验证。
- 不引入新的服务端 API，但会影响 UI 行为、视觉表现和部分现有设置项的承载方式。
